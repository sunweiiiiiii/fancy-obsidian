
# 项目架构文档：基于移动端的 2D 面部医美仿真与网格联动系统
## 1. 项目概述与技术栈
本项目旨在安卓移动端构建一套高保真、零延迟的**医美术后效果仿真编辑系统**。系统通过在用户真实面部照片上叠加带有科技感的 2D 可视化网格，允许医生通过手势拖拽网格节点，实时、平滑地改变面部软组织形态（如瘦脸、下颌缘提升、开眼角等），实现“所见即所得”的医美沟通方案。
 * **目标硬件：** 高通骁龙 8 至尊版 (Snapdragon 8 Elite) 安卓设备。
 * **前端与交互：** Tauri (跨平台框架) / 原生 Android UI。
 * **核心渲染 API：** OpenGL ES 3.2 或 Vulkan (实现极低延迟的网格重绘)。
 * **视觉与 AI 算法：** Google MediaPipe Face Mesh (478 关键点)、MediaPipe Selfie Segmentation (人像抠图)。
 * **底层计算引擎：** C++ 与 OpenCV (NDK 交叉编译，负责矩阵解算与形变映射)。
## 2. 核心架构设计：分层渲染与隔离联动
为了彻底解决复杂环境背景在形变时发生的“扭曲穿帮”问题，以及强化医疗软件的专业感，渲染管线采用**绝对分层策略**：
 1. **背景层 (Background Layer) - 绝对静止：**
   利用 Selfie Segmentation 剔除原图杂乱背景。底层画板强制替换为纯净的医疗级深灰色/黑色底色（配合 Dark Mode UI）。该层在渲染期间被完全锁死。
 2. **前景层 (Foreground Layer) - 联动形变：**
   仅保留带 Alpha 透明通道的人像（含面部、头发、躯干）。
 3. **网格控制层 (Mesh Control Layer) - 交互锚点：**
   在前景层上方生成由 MediaPipe 关键点构成的 2D 拓扑三角形网格。前景层的人像像素作为**纹理 (Texture)** 绝对绑定在这张网格上。
## 3. 核心数据管线 (Data Pipeline)
### 3.1 输入与前置对齐 (Preprocessing)
 1. 采集高清正面照片。
 2. MediaPipe 提取 478 个特征点，计算双眼瞳孔中心的连线角度，利用 OpenCV 的仿射变换将图像进行“绝对正位矫正”。
 3. 执行背景剥离，生成纯净的前景人像贴图。
### 3.2 拓扑构建与纹理映射 (Topology & Mapping)
 1. **坐标归一化：** 将 MediaPipe 的绝对像素坐标转换为 0 \sim 1 的归一化坐标 (u, v)，统一渲染比例。
 2. **三角剖分：** 利用德劳内三角剖分 (Delaunay Triangulation) 算法，或直接沿用标准 3DMM 的面部连接表，将关键点连成极其致密的 2D 三角形网格。
 3. **UV 绑定：** 在 OpenGL 中，将抠出的人像前景作为贴图，根据顶点的 (u, v) 坐标，完成图像像素到三角形网格的填充渲染。

### 3.4 极速渲染管线 (Render Loop)
解算完成的全新顶点数组流向 GPU。OpenGL/Vulkan 触发重绘指令，通过硬件级的**双线性插值 (Bilinear Interpolation)** 瞬间完成包含几十万像素的局部纹理拉伸。由于骁龙 8 至尊版的单帧吞吐量极高，该过程耗时须控制在 2\text{ms} 内，实现 120Hz 丝滑拖拽。
## 4. UI/UX 与“科技感”视觉规范
 * **深色医疗极简主义：** 去除多余边框。除底部抽屉式的工具栏外，屏幕 85% 以上空间留给纯黑底色的高保真人像。
 * **高定科技网格：** 覆盖在面部的 2D 控制网格，切忌使用实心或高对比度线条。采用冷青色 (Cyan) 或银灰色，设置透明度（Alpha 值为 0.3-0.5），线宽 1\text{px}。
 * **Shader 光效加持：** 在 OpenGL 渲染网格时，启用多重采样抗锯齿 (MSAA)，并为线条加入微弱的泛光效果 (Bloom / Glow Shader)。当手指触控某区域时，该局部网格的亮度瞬间提升，松手后缓慢衰减，提供极致的互动反馈。
## 5. Agent 开发路径与阶段里程碑 (Implementation Phases)
### Phase 1: 图形渲染管线闭环 (脱离 AI 的纯 C++ / OpenGL 测试)
 * **任务：** 不接入摄像头和 MediaPipe。将一个开源的标准面部 .obj 文件（如来自 Free3D 的低模数据，包含 v 和 f）硬编码加载进内存。将一张测试照片作为纹理贴上。
 * **目标：** 跑通 OpenGL 的 UV 贴图逻辑，实现基础的手势识别、矩阵解算、以及无穿破的网格拖拽重绘。确立纯图形计算的性能基准帧率。
### Phase 2: AI 视觉管线接入 (MediaPipe 融合)
 * **任务：** 接入摄像头流或本地图片。跑通 MediaPipe C++ / NDK 接口。用实时提取的 478 个特征点动态替换 Phase 1 中的 .obj 顶点数据。
 * **目标：** 实现照片导入 -> 自动抠图变黑底 -> 自动生成贴合五官的动态科技网格。
### Phase 3: 医美业务逻辑封装
 * **任务：** 限制“全局自由乱拉”的权限。开发特定的医美工具包，例如“下颌缘收紧刷”、“开眼角拉杆”。
 * **目标：** 将底层的顶点数学变换，封装为前端医生可用的高频业务操作接口，完成最终产品的商业化闭环。
